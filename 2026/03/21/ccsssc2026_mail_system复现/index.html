ccsssc2026_MailSystem复现

题目分析

保护全开，沙箱禁了execve和execveat

一个邮件系统 PWN 题，程序提供用户注册/登录、收发邮件功能，以及管理员转发邮件等操作。

  • 普通用户菜单：写草稿、读收件箱、发信、登出
  • 管理员菜单：改用户、删用户、代发邮件、用户间转发邮件

用户结构体逆向

分析方法概述

逆向分析结构体的核心方法是： 追踪函数中对结构体成员的访问模式 。

在 IDA 反编译代码中，结构体成员的访问通常表现为：

  • ptr + offset ：字节偏移访问

  • ptr[index] ：当 ptr 是 size_t* 时，实际偏移是 index * 8

admin_check用strncmp作比较，该函数的特点是遇到\x00就会截止，故可以输入\x00绕过检验（大约1/127）概率。

下界检查缺失

在sub_2cd6(用于实现admin mail user to user)中，对两个user的id均只检查了上界


实际上，联系0x7060附近的结构，我们可以把std_err和std_out等当成用户来写

off-by-one获取admin权限

create_user中user_id理应只有12个，即0-11，检查却是0-12，从数据结构中不难看出，p_user[12]就是admin_info，当前12个user被填满时，再输入即可通过memset清空admin的结构体内容，之后只需两个\x00即可通过strcmp的验证

漏洞利用

辅助函数

  • 注册用户函数
  • 选择菜单选项 2（注册）
  • 输入用户名和密码
  • 登录用户函数
  • 选择菜单选项 1（登录）
  • 输入用户名和密码
  • 写邮件/草稿函数
  • 选择菜单选项 1（写邮件）
  • 指定写入的字节数（1-256）
  • 发送邮件内容
  • 发送 5 封邮件触发封禁函数
  • 循环 5 次：
    1. 写一个 1 字节的草稿（内容为 “A”）
    2. 选择菜单选项 3（发送邮件）
    3. 指定目标用户 ID
  • 处理可能的覆盖提示或封禁提示
  • 目的 ：在 10 秒内发送 5 封邮件，触发频率限制封禁机制

主流程

流程图

Step1 admin权限获取

注册 8 个用户

  • 注册用户 u1-u8，密码分别为 p1-p8
  • 此时 slots 0-7 被占用
  • 下一个要注册的用户索引从 9 开始

封禁用户并填补槽位

  • 关键步骤！ 循环 5 次（i = 1, 2, 3, 4, 5）：
    1. 登录用户 ui
    2. 发送 5 封邮件给 user 8，触发封禁
    3. 注册新用户填补空位（保持 active_count ≤ 7）
      执行后的状态 ：

登录admin

Step2 libc泄露

  • 选择 admin 菜单选项 4（Mail user to user）
  • 关键 ：source ID = -3（数组越界！）
    • users[-3-1] = users[-4] = *(0x7040) = stderr
    • 程序把 stderr 当作用户结构体
  • dest ID = 8（正常用户 u8）
  • 处理可能的覆盖提示
  • 选择转发草稿（选项 1）
  • 效果 ：复制 stderr + 0x110 （”草稿区”）到 user 8 的收件箱
  • stderr + 0x110 包含 libc 内部指针，可以泄漏 libc 基址

读取泄漏数据并计算 libc 基址

  • 登出 admin
  • 登录 user 8
  • 读取收件箱（菜单选项 2，然后选择读取邮件）
  • 读取泄漏的数据
  • u64() ：将 8 字节转换为整数
  • 计算公式 ： libc_base = leak - 131 - IO_2_1_stdout
    • leak 是 IO_2_1_stdout + 131 的地址
    • 减去偏移得到 libc 基址
  • 退出读取邮件菜单

Step3 FSOP

构造payload

  • 计算各种 gadget 和地址：
    • stdout_ptr ：stdout 的 FILE 结构体地址
    • setcontext_addr ：setcontext 函数 + 61 偏移（用于栈迁移）
    • magic_gadget ：关键的 magic gadget
- 各种 pop gadget：用于构造 ROP 链
- ROP_DEST ：ROP 链存放的目标地址（bss 段 + 0x800）
  • 构造伪造的 _IO_FILE 结构体 （256 字节）
  • 关键字段解释：
    • _flags = 0 ：清除 _IO_NO_WRITES 标志
    • _IO_write_base = _IO_write_ptr = 0 ：触发 overflow 逻辑
    • vtable = _IO_wfile_jumps ：劫持 vtable
    • _wide_data ：指向伪造的 _wide_data 结构体
    • doallocate = magic_gadget ：伪造的 doallocate 函数指针
    • setcontext 地址：用于栈迁移
    • rsp = ROP_DEST, rip = read ：栈迁移后执行 read
      攻击链 ：
  • 将 payload 写入 user 8 的草稿

触发 FSOP

  • 登出 user 8
  • 再次以 admin 身份登录（使用 \x00 绕过）
  • 关键 ：将 user 8 的草稿转发到 dest ID = -7
    • users[-7-1] = users[-8] = *(0x7020) = stdout
    • 程序将 payload 复制到 IO_2_1_stdout 结构体
  • 效果 ：覆写 stdout 的 FILE 结构体，植入恶意 payload
  • 等待 0.5 秒，确保操作完成

Step4 ORW链

  • 构造 ORW ROP 链
  • 第一部分： open(“/flag”, 0)
    • pop rdi; ret → RDI = “/flag” 字符串地址
    • pop rsi; ret → RSI = 0 (O_RDONLY)
    • call open
  • 第二部分： read(3, buf, 0x100)
    • pop rdi; ret → RDI = 3 (文件描述符)
    • pop rsi; ret → RSI = buf 地址
    • pop rdx; pop rbx; ret → RDX = 0x100, RBX = 0
    • call read
  • 第三部分： write(1, buf, 0x100)
    • pop rdi; ret → RDI = 1 (stdout)
    • pop rsi; ret → RSI = buf 地址
    • pop rdx; pop rbx; ret → RDX = 0x100, RBX = 0
    • call write
  • 最后调用 exit 退出
  • 将 ROP 链填充到 0x100 字节
  • 在 0x100 偏移处添加 “/flag” 字符串
  • 发送 ORW ROP 链
  • 进入交互模式，等待 flag 输出
